Pourquoi construire un site statique



...Mais nous on aime les sites statiques.

Un site statique génère des pages telles qu'elles se présentent dans l'espace où elles sont conservées, au contraire d'un site dynamique qui utilise une application web pour générer des pages différentes d'un utilisateur à l'autre. 

Hugo, Jekyll, Netlify et Mkdocs sont quelques générateurs de sites statiques. 
Ces logiciels permettent de générer des sites web à partir de répertoires hébergés par exemple sur github ou gitlab. 

Publii donne la possibilité de déployer un site statique depuis Gitlab ou Github sans connaître Git au moyen d'une interface graphique de type CMS. 

Cette solution requiert tout de même d'être en mesure de déployer des pages depuis gitlab ou de savoir utiliser github pages. Une solution utile pour des chercheurs et des chercheuses qui n'auraient pas le temps de se former à la création de sites web avec des moteurs de site utlisant des fichiers texte (type jekyll, mkdocs, Hugo, Quarto, etc.)
